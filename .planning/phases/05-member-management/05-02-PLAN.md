    Create `frontend/src/components/members/InviteModal.tsx`:
    - Props interface: `{ open: boolean; onOpenChange: (open: boolean) => void }`.
    - Zod schema:
      ```typescript
      const inviteSchema = z.object({
        email: z.string().email("Valid email required"),
        role: z.enum(["ADMIN", "MEMBER"]),
      })
      type InviteFormValues = z.infer<typeof inviteSchema>
      ```
    - `useForm<InviteFormValues>({ resolver: zodResolver(inviteSchema), defaultValues: { email: "", role: "MEMBER" } })`.
    - `const queryClient = useQueryClient()`.
    - Mutation:
      ```typescript
      const inviteMutation = useMutation({
        mutationFn: (values: InviteFormValues) => inviteMember(values),
        onSuccess: () => {
          queryClient.invalidateQueries({ queryKey: ["members"] })
          form.reset()
          onOpenChange(false)
        },
      })
      ```
    - Render a shadcn `Dialog` controlled by `open` / `onOpenChange`, with `DialogContent` containing `DialogHeader` ("Invite Member" title, description "Send an invitation email to a new teammate."), the `Form` with an email `FormField` (Input) and a role `FormField` rendered as a shadcn `Select` with items ADMIN and MEMBER (NOT OWNER), `FormMessage` for errors, and a `DialogFooter` with a Cancel button (calls `onOpenChange(false)`) and a submit Button.
    - Submit button disabled while `inviteMutation.isPending`.
    - On mutation error, show the error text inside the dialog (e.g. a small red `<p>` reading `inviteMutation.error` message; 409 conflict surfaces as "already invited" — just show the server message).

    Create `frontend/src/components/members/InviteModal.test.tsx`:
