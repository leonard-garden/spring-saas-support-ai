<objective>
Add a password show/hide toggle to every password input across the auth pages. The toggle is implemented inline (not a separate reusable component — keep it simple for v0.2) using a `useState<boolean>` flag and the `Eye` / `EyeOff` icons from `lucide-react` (already a transitive dep via shadcn). When toggled, the input `type` switches between `"password"` (masked, default) and `"text"` (visible).

Purpose: Demo-readiness polish — recruiters expect modern auth UX. Note: this plan does not introduce any new requirement; KB-02 was already satisfied in 06-01. This is purely the "Polish" half of Phase 6 success criterion #3 from ROADMAP.md.

Output: LoginPage.tsx, SignupPage.tsx, and ResetPasswordPage.tsx (if it exists) updated with password visibility toggles. No new components, no new dependencies.
</objective>

```typescript

  <Input type={show ? "text" : "password"} {...field} className="pr-10" />

    className="absolute right-2 top-1/2 -translate-y-1/2 ...">
    {show ? <EyeOff className="h-4 w-4" /> : <Eye className="h-4 w-4" />}
  </button>
</div>
```
