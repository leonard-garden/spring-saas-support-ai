    Operator steps (Render dashboard):

    1. Open the backend service `spring-saas-support-ai` in the Render dashboard.
    2. Go to **Environment** → find `CORS_ALLOWED_ORIGINS`.
    3. Update its value to a comma-separated list that INCLUDES the Render frontend URL from Plan 07-01. Use this exact pattern (substitute the real frontend URL):

       ```
       http://localhost:5173,https://support-saas-frontend.onrender.com
       ```

       Rules:
       - Include the scheme (`https://`).
       - No trailing slash on any origin.
       - Keep `http://localhost:5173` so local dev still works.
       - Separate with commas, no spaces.
    4. Save. Render will prompt to redeploy the backend — accept (or click **Manual Deploy → Deploy latest commit**).
    5. Wait for the backend deploy to finish (status returns to "Live", `/actuator/health` returns 200).
    6. Smoke ping from a terminal (optional but recommended):

       ```bash
       curl -i -X OPTIONS https://spring-saas-support-ai.onrender.com/api/v1/auth/login \
         -H "Origin: https://support-saas-frontend.onrender.com" \
         -H "Access-Control-Request-Method: POST"
       ```

       Expect `HTTP/2 200` (or 204) and `access-control-allow-origin: https://support-saas-frontend.onrender.com` in the response headers.

    Resume signal: Operator replies "CORS updated, backend redeployed" and pastes the preflight response status line.

    Step S1 — Loads cleanly:
      - Frontend URL returns 200, app shell renders, zero CORS errors in console.

    Step S2 — Hard-refresh routing:
      - Navigate to `/login`. Press F5. App re-renders (not a Render 404). Repeat for one more deep route (e.g. `/dashboard` — expect redirect to `/login` if unauth, which still proves the SPA shell loaded).

      - Log out if needed, then log in with the credentials from S3.
      - Expect: 200 from `POST /api/v1/auth/login`, refresh-token cookie/storage set, redirect to `/dashboard`.

    Step S5 — Dashboard:
      - `/dashboard` renders with business name, plan indicator, stat cards, sidebar (per Phase 4 success criteria).

    Step S6 — Invite a member:
      - Open the members page, click invite, enter an email like `invitee+<timestamp>@example.com`, submit.

    Step S7 — Logout:
      - Click logout. Expect: redirect to `/login`, refresh token revoked server-side, access token cleared. Manually visit `/dashboard` → redirect back to `/login` (proves session is gone).

    For each step record: PASS / FAIL / status code / any console errors. If any step fails with CORS, double-check Task 1 (the origin value must exactly match the browser-bar origin, including scheme).
