<objective>
Install vitest test infrastructure, define shared auth TypeScript contracts, and extend the existing axios instance with a refresh-lock response interceptor + request token injection. This is the foundation layer Phase 3 (auth pages) and all later phases will consume.

Purpose: Without a refresh lock, concurrent API calls on app mount each trigger a /auth/refresh, invalidate each other's tokens, and cascade to logout. The lock guarantees exactly one refresh in flight at any time. Without a test framework, the validation contract for this phase cannot be met.

Output: Working `npm run test -- --run`, populated test stubs, an `api.ts` that injects bearer tokens and recovers from 401s by refreshing once and replaying the original request.
</objective>

    2. Implement `frontend/src/lib/api.ts` (GREEN) — extend the existing file. Keep the existing `axios.create` call and exports unchanged at the top. Add:

       ```ts
       import axios, { AxiosError, InternalAxiosRequestConfig } from "axios"
       import type { ApiResponse, AuthResponse } from "@/types/auth"
       import { getRefreshToken, setRefreshToken, clearRefreshToken } from "@/lib/tokenStorage"
       import { useAuthStore } from "@/store/authStore"  // referenced; resolved at runtime

       // ... existing baseURL guard + api = axios.create({...}) stays as-is

       // Module-level lock — exactly one refresh in flight
       let refreshPromise: Promise<string> | null = null

       async function doRefresh(): Promise<string> {
         const refreshToken = getRefreshToken()
         if (!refreshToken) throw new Error("No refresh token")

         // Use raw axios.post (NOT the `api` instance) to avoid interceptor recursion — see RESEARCH.md Pitfall 1
         const res = await axios.post<ApiResponse<AuthResponse>>(
           `${baseURL}/api/v1/auth/refresh`,
           { refreshToken },
           { headers: { "Content-Type": "application/json" } }
         )
         if (!res.data.success || !res.data.data) {
           throw new Error(res.data.error ?? "Refresh failed")
         }
         const { accessToken, refreshToken: newRefreshToken } = res.data.data
         setRefreshToken(newRefreshToken)
         return accessToken
       }

       // Request interceptor — inject bearer token
       api.interceptors.request.use((config: InternalAxiosRequestConfig) => {
         const token = useAuthStore.getState().accessToken
         if (token) {
           config.headers.set("Authorization", `Bearer ${token}`)
         }
         return config
       })

       // Response interceptor — refresh-and-retry on 401
       api.interceptors.response.use(
         (response) => response,
         async (error: AxiosError) => {
           const originalRequest = error.config as (InternalAxiosRequestConfig & { _retry?: boolean }) | undefined
           if (!originalRequest || error.response?.status !== 401 || originalRequest._retry) {
             return Promise.reject(error)
           }
           originalRequest._retry = true

           try {
             if (!refreshPromise) {
               refreshPromise = doRefresh().finally(() => { refreshPromise = null })
             }
             const newAccessToken = await refreshPromise
             const store = useAuthStore.getState()
             if (store.user) {
               store.setAuth(newAccessToken, store.user)
             }
             originalRequest.headers.set("Authorization", `Bearer ${newAccessToken}`)
             return api(originalRequest)
           } catch (refreshErr) {
             clearRefreshToken()
             useAuthStore.getState().clearAuth()
             return Promise.reject(refreshErr)
           }
         }
       )
       ```

       Notes:
       - The `useAuthStore` import resolves to the store file created in plan 02-02. Tests mock it. At runtime, Vite/TS will resolve the import once that file exists.
       - Do NOT remove the existing `export default api` and `export const api`.
